# 启动与测试计划

我们将按照以下步骤启动项目并验证核心功能：

## 1. 环境准备
- **安装依赖**：运行 `npm install` 以确保所有新添加的库（如 `@react-native-async-storage/async-storage` 和 `react-native-draggable-flatlist`）都已正确安装。

## 2. 启动服务
- **启动开发服务器**：运行 `npm start`。这将启动 Expo 开发服务器。
- **Web 预览**：我们将优先使用 Web 端预览（按 `w`）来快速验证流程，或者提供二维码供手机端 Expo Go 测试。

## 3. 功能验证流程
我们将验证 MVP 的三个核心模块：
1.  **作业设定 (Setup)**：
    - 添加 2-3 个作业项（测试名称长度限制、时间范围）。
    - 尝试拖拽排序。
    - 尝试修改时长和删除作业。
2.  **专注执行 (Focus)**：
    - 开始第一个作业，观察计时器。
    - 模拟“暂停”操作，验证中断次数增加。
    - 完成当前作业，进入下一项。
    - 结束当日作业。
3.  **家长报告 (Report)**：
    - 检查今日总时长、中断次数、完成率是否与刚才的操作一致。
    - 切换日期查看（验证数据持久化）。
    - 检查周汇总数据显示。
